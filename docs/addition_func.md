# [DoTo MVP] IA 반영 수정 요청서 + 작업 티켓(v0.1)


수신: CTO  
목적: IA/와이어프레임 정합성 확보 + “투두앱처럼 보이는 문제” 해소 + 핵심 루프(오늘퀘스트→완료→변화) 최소 구현

---

## 0) 확정 의사결정(전제)
1) **TODOS(할일) 탭 삭제**
- ‘할일’은 별도 개인 투두로 분리하지 않고 **Quest(퀘스트)로 통일**합니다.
- 즉, “할일=퀘스트(습관/과업)”입니다.

2) **캐릭터는 Bottom Tab에서 제외**
- MVP처럼 **상단 캐릭터 영역(CP-04) 클릭 → 캐릭터 화면(APP-P04) 진입** 흐름으로 갑니다.

3) **보상 구조(상세 룰)는 보류**
- 상점/진화/연속 보상 등 **복잡한 룰은 추후 확정**합니다.
- 단, MVP 설득을 위해 “완료 후 변화”는 **최소로만 연결**(XP/코인 고정값 가산 OK)

---

## 1) IA Screen ID(참고)
- Onboarding: OB-P01~03  
- App Main: APP-P01(비전/목표) / APP-P02(프로젝트) / APP-P03(오늘퀘스트) / APP-P04(캐릭터)  
- Project Detail: APP-P07  
- Overlay: OV-02(준비중 잠금) / OV-03(구조 도움말)  
- Modal: MD-01(완료확인 Optional) / MD-02(보상팝업)

---

## 2) 핵심 문제(현재 MVP에서 체감되는 이슈)
- 목표/프로젝트/오늘이 모두 체크박스 형태로 보여 **투두앱처럼 보임**
- 프로젝트 생성에서 **퀘스트(습관/과업)를 정할 수 없음**
- 프로젝트 클릭 시 **상세(APP-P07)로 진입/조회가 안 됨**
- 캐릭터는 탭보다 **상단 클릭 흐름이 더 자연스러움**
- 보상 구조는 지금 확정하기 어려우니 **보류(단, 변화 최소 연결 필요)**

---

## 3) 화면/용어 정의(사용자 혼란 방지용 문구)
### 3-1) 구조 정의(OV-03에 노출)
- **Goal(3개월)**: 3개월 후 도달할 **측정 가능한 결과**
- **Project(4주)**: Goal을 위한 **4주 실행 묶음**
- **Quest(일)**: 오늘 수행하는 **습관/과업 단위(완료 체크는 여기서만)**

### 3-2) 구조 예시(OV-03 또는 빈 화면 가이드에 사용)
- **목표(Goal, 3개월)**: 5kg 감량하기  
- **프로젝트(Project, 4주)**: 헬스 루틴 만들기  
- **퀘스트(과업/일회성)**: 헬스장 등록하기  
- **퀘스트(습관/반복)**: 주 3회 운동하기  

(추가 예시)
- 목표: 3개월 내 OPIc IM2 달성  
- 프로젝트: 4주 스피킹 루틴 만들기  
- 퀘스트(과업): OPIc 교재 1권 구매하기  
- 퀘스트(습관): 평일 10분 말하기

### 3-3) 온보딩/설명 문구(짧게)
- 슬로건(택1)
  1) **목표를 세우고, 오늘의 행동으로 캐릭터를 성장시켜요.**
  2) **목표 → 프로젝트 → 오늘 퀘스트. 오늘 한 걸음이 성장으로 이어져요.**

# 4) 작업 티켓(Backlog) — 우선순위/완료조건 포함

## P0 (이번 스프린트 필수)
### T1) Bottom Tab 재구성
- 내용: **TODOS 탭 삭제**, **캐릭터 탭 제거**, 탭=APP-P01/APP-P02/APP-P03만 유지
- 완료조건: 하단 탭에 `비전/목표`, `프로젝트`, `오늘퀘스트`만 보임

### T2) 캐릭터 진입 동선 변경(상단 클릭)
- 내용: 상단 캐릭터 영역(CP-04) 클릭 → **APP-P04 이동**
- 완료조건: 탭 없이 캐릭터 화면 진입 가능

### T3) UI 교정(“투두앱처럼 보이는 문제” 해결)
- 내용:
  - **APP-P01/APP-P02는 카드형 UI(기간/진행률/요약)**  
  - 체크박스(완료)는 **APP-P03에서만** 노출
- 완료조건: 목표/프로젝트 화면에 체크리스트 UI가 없음

### T4) 프로젝트 생성/수정에서 퀘스트 추가 가능
- 내용:
  - 프로젝트 생성/수정 플로우에서 **퀘스트 추가**
  - 타입: **습관(반복)** / **과업(일회성)**
  - 최소 필드: 타입, 제목, (습관이면 반복 설정: 요일/주 n회 등)
- 완료조건: 프로젝트에 퀘스트(습관/과업)를 추가 저장 가능

### T5) 프로젝트 상세 진입 구현(APP-P07)
- 내용: 프로젝트 카드 클릭 → **APP-P07**
- APP-P07 최소 구성:
  - 프로젝트 요약(기간/진행률)
  - 퀘스트 리스트(습관/과업)
  - “오늘퀘스트로 이동” 링크(→ APP-P03)
- 완료조건: 프로젝트 클릭 시 상세 화면에서 내용 확인 가능

---

## P1 (가능하면 이번 스프린트)
### T6) OV-03 구조 도움말 적용(정의+예시)
- 내용: Goal/Project/Quest 정의 + 예시를 **Overlay(OV-03)**로 제공(페이지 추가 X)
- 완료조건: 사용자가 화면에서 정의를 확인 가능

### T7) 오늘퀘스트 완료 → 최소 변화 연결(보상 룰은 보류)
- 내용:
  - APP-P03 완료 체크 → (선택) MD-01 → **MD-02**
  - MD-02는 최소 형태로 XP/코인 가산(고정값 OK)
  - 상단(CP-04)에 XP/레벨 변화 즉시 반영
- 완료조건: “완료→변화”가 사용자에게 바로 보임
- 주의: 연속/진화/상점 등 **보상 상세 룰은 추후 확정**, 지금은 단순 가산만 유지

---

## P2 (후순위)
### T8) 빈 화면(Empty state) CTA/문구 보완
- 내용: 목표/프로젝트/오늘퀘스트 빈 상태에서 “다음 행동” 가이드 강화
- 완료조건: 빈 화면에서 사용자가 무엇을 해야 할지 명확
